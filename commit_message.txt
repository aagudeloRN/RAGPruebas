feat: Implementación de arquitectura multi-tenant y correcciones

Este commit introduce una arquitectura multi-tenant completa, permitiendo la gestión de múltiples Bases de Conocimiento (KB) aisladas.

Mejoras y Características:
- Refactorización completa para soportar múltiples KBs, con la KB activa gestionada a través de la variable de entorno `ACTIVE_KB_ID`.
- Nuevo panel de administración en `/admin` para crear y activar KBs.
- El pipeline de ingesta ahora procesa documentos grandes en lotes para evitar errores de API en Pinecone.
- Todas las operaciones de datos (CRUD, RAG, etc.) ahora operan en el contexto de la KB activa, asegurando el aislamiento de los datos.

Correcciones de Bugs:
- Se corrigió un `AttributeError` en la eliminación de documentos después de renombrar la columna `preview_image_url` a `cover_image_url`.
- Se solucionó un `TypeError` en el chat de conocimiento al no pasar el `kb_id` a los servicios RAG.
- Se corrigió un `ImportError` al renombrar `get_kb` a `get_knowledge_base` para consistencia.
- Se ajustó el prompt del RAG para asegurar que las respuestas sean siempre en español y se ciñan al contexto.
- Se redujo el umbral de relevancia de Cohere para mejorar la recuperación de respuestas.
